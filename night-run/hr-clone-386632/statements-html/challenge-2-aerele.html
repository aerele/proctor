Ravi works in a warehouse and loves building neat towers of boxes. He has a row of N boxes, and he will try to stack them into a single tower, going through the boxes strictly from left to right.

He starts with an empty tower. For each box in order, he looks at its width: - If the tower is empty, he places the box (it becomes the bottom). - Otherwise, he places the box on top ONLY IF its width is less than or equal to the width of the box currently on top. A wider box would be unstable, so he simply throws that box away and moves on.

For example, with widths [5, 3, 4, 2, 1]: he places 5, then 3 (3 <= 5), throws away 4 (4 > 3), places 2 (2 <= 3), places 1 (1 <= 2). The final tower has 4 boxes.

Given the widths of the boxes in order, find how many boxes end up in Ravi's tower.

Input Format

The first line contains an integer N, the number of boxes. The second line contains N space-separated integers, the widths of the boxes in order from left to right.

Constraints

1 <= N <= 100000 1 <= width of each box <= 1000000000

Output Format

A single integer: the number of boxes in the final tower.

Sample Input 0

5
5 3 4 2 1

Sample Output 0

4

Explanation 0

Place 5, place 3 (3<=5), discard 4 (4>3), place 2 (2<=3), place 1 (1<=2). Tower has 4 boxes.
