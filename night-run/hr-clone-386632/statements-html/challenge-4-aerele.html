A water tanker must deliver water to N houses standing in a row, numbered 1 to N from left to right. House i needs exactly a[i] litres.

The tanker starts at a depot on the far left, FULL with a capacity of C litres, and drives rightward. At each house it tries to pour out exactly the amount that house needs. It will only serve a house if it currently holds at least that much water; otherwise it first drives back to the depot, refills completely to C litres, returns, and then serves the house. The tanker always serves houses strictly in order 1, 2, ..., N.

For example, with needs [3, 4, 3, 2] and capacity C = 6: it serves house 1 (3 left), cannot serve house 2 (needs 4 > 3) so it refills (1st refill) then serves it (2 left), cannot serve house 3 (needs 3 > 2) so it refills (2nd refill) then serves it (3 left), and finally serves house 4 (1 left). That is 2 refills.

The tanker starts full, so the initial fill does not count as a refill. Count only the mid-route refills.

Input Format

The first line contains two integers N and C: the number of houses and the tanker capacity. The second line contains N space-separated integers a[1], a[2], ..., a[N], the litres needed by each house.

Constraints

1 <= N <= 100000 1 <= a[i] <= C <= 1000000000

Output Format

A single integer: the number of refills the tanker makes (not counting the initial fill).

Sample Input 0

4 6
3 4 3 2

Sample Output 0

2

Explanation 0

Start full (6). Serve 3 (3 left). House 2 needs 4 > 3: refill (1), serve (2 left). House 3 needs 3 > 2: refill (2), serve (3 left). Serve 2 (1 left). Total 2 refills.
